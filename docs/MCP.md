# MCP-сервер EdgeLab Space

EdgeLab Space публикует библиотеку сообщества как MCP-сервер. Подключите его в
любом MCP-клиенте — Claude Code, cowork, Hermes, Cursor — и ваш агент получит
прямой доступ к навыкам, урокам, разборам и вопросам сообщества, не открывая
интерфейс.

## Подключение

Два способа — выбирайте по клиенту.

### Способ 1 — кастомный коннектор (OAuth, без ручного ключа)

Для **Claude.ai** (веб и десктоп) и **Claude Cowork**. Добавляете EdgeLab Space
как кастомный коннектор по одному URL — ключ вводить не нужно, доступ
подтверждается входом через Telegram.

1. Настройки → «Коннекторы» → «Добавить кастомный коннектор».
2. URL: `https://mcp.edgelab.space/mcp`
3. Клиент сам зарегистрируется (OAuth 2.1, Dynamic Client Registration) и откроет
   вход. Войдите через бота `@edgelabspacebot` и нажмите «Подтвердить».
4. На экране согласия разрешите доступ — клиент получит токен и подключится.

Доступ привязан к вашей подписке: при отмене подключение перестаёт работать.
Токены хранятся на сервере в зашифрованном виде; подключение можно отозвать.

### Способ 2 — персональный ключ (Bearer)

Для **Claude Code, Codex, Cursor** и обычного `curl` — там, где коннектор
задаётся конфигом.

| Параметр | Значение |
|---|---|
| URL | `https://mcp.edgelab.space/mcp` |
| Транспорт | Streamable HTTP (stateless, JSON-ответы) |
| Аутентификация | `Authorization: Bearer edgelabspace_...` |

Персональный ключ формата `edgelabspace_...` выпускается в кабинете на
`platform.edgelab.space/agent-identity`. Ключ показывается один раз — сохраните
его. Доступ действует, пока активна ваша подписка; при отмене сервер начинает
отвечать `Unauthorized`.

## Инструменты

Все инструменты только для чтения и возвращают материалы, доступные вашей
когорте.

| Инструмент | Назначение |
|---|---|
| `list_library(category?, limit)` | Список материалов библиотеки (category: lesson / skill / usecase / live / workshop / guide) |
| `search_library(query, limit)` | Поиск по библиотеке по названию и описанию |
| `get_material(slug)` | Полный материал: цель, результат, статья, копируемый промпт скилла, видео, транскрипт (поле `transcript`: таймкодные главы + полный вербатим после сентинела `<!-- EDGELAB:FULL_TRANSCRIPT:v1 -->`), шаги, дочерние уроки |
| `list_hub_questions(query?, category?, limit, offset)` | Лента вопросов сообщества (AgentTinder) |
| `get_hub_question(question_id)` | Полный вопрос + источники, файлы и комментарии/ответы |
| `get_my_audience_profile()` | Текущий профиль онбординга + отдельная диагностика «Точка А» |
| `get_my_feedback(limit)` | ВАШИ СОБСТВЕННЫЕ оценки уроков и ответы NPS (новые первыми, до 100) |
| `list_digests(period?, date?, from_?, to?, last_n_days?)` | Лёгкий индекс дайджестов сообщества за период (без тел) — найти, что есть |
| `get_digest(period?, date?, from_?, to?, last_n_days?)` | Дайджест(ы) сообщества за период из накопленной истории |

### Профиль аудитории

`get_my_audience_profile()` возвращает независимые блоки:

```json
{
  "profile": {"source": "space_onboarding", "answers": {"q1": "..."}, "completed_at": "2026-07-12T12:00:00Z"},
  "diagnostics": {"source": "tochka_a", "created_at": "2026-06-01T09:00:00Z", "taken_at": "2026-06-01T09:00:00Z"},
  "quiz": {"source": "tochka_a", "created_at": "2026-06-01T09:00:00Z", "taken_at": "2026-06-01T09:00:00Z"}
}
```

`quiz` — deprecated alias и всегда равен `diagnostics`. Возможны все четыре
состояния: оба блока заполнены, только `profile`, только `diagnostics`, оба
`null`. Частичный онбординг остаётся `profile` и не смешивается с диагностикой.

### Ваш фидбэк

`get_my_feedback(limit=100)` возвращает ВАШИ СОБСТВЕННЫЕ оценки уроков и ответы
NPS — только ваши данные (member scope). Список отсортирован от новых к старым и
подрезается до `limit` (1..100). Каждая строка: `knowledge_item_id` (`null` для
платформенного NPS-ответа), `metric_type` (`lesson_score` / `nps`), `score`
(0..10), `comment`, `created_at`, `updated_at`.

### Hub: строгая валидация и безопасные ошибки

`category` нормализуется (`strip` + lowercase) и должен быть одним из:
`devops`, `контент`, `крипто`, `маркетинг`, `дизайн`, `код`,
`юридическая тема`, `reels`, `ревью`. Неизвестное или пустое переданное значение
возвращает validation error и не расширяет выдачу. `question_id` должен быть UUID.
Временные сбои данных возвращаются как стабильная публичная ошибка без внутренних
URL, SQL/PostgREST-параметров или содержимого ответа.

Архивированные вопросы, их комментарии и производные дайджесты не возвращаются
ни списком, ни по прямому ID.

### Дайджесты сообщества

Дайджест — периодическая AI-сводка того, что произошло в клубе (новые материалы,
заметные вопросы Hub, темы чата). История копится (append-only), её можно смотреть
за любой срок до 120 дней. Доступно членам с тарифом **edge** _(функция в
постепенном запуске — пока может отвечать «выключено»)_.

Укажите ровно одну спецификацию периода (без неё — `today`):

| Спецификация | Пример | Окно (МСК) |
|---|---|---|
| `period` | `period="today"` / `"yesterday"` / `"this_week"` / `"last_week"` | именованный период |
| `date` | `date="2026-06-12"` | один московский день |
| `from_` + `to` | `from_="2026-06-01", to="2026-06-10"` | произвольный диапазон `[from, to)`, до 120 дней |
| `last_n_days` | `last_n_days=30` | последние N дней `[now-N, now)`, 1..120 |

`list_digests` сначала показывает доступные дайджесты (метаданные + заголовок),
затем `get_digest` отдаёт тело. `get_digest` возвращает
`{status, count, preview_count, final_count, truncated, digests:[…]}`; для ещё
не закрытого текущего периода отдаётся последний предварительный вариант.

## Founder-инструменты (только для основателя)

Эти инструменты требуют **founder scope** — они доступны только member-токену,
чей id входит в список основателей. Обычный участник получает ошибку `forbidden:
founder scope required`, инструмент для него не работает. Они отдают
**агрегаты по всему сообществу** и карточку отдельного участника, поэтому здесь
описаны для полноты, но недоступны рядовому агенту.

| Инструмент | Назначение |
|---|---|
| `audience_summary()` | Агрегаты аудитории: обязательный онбординг (`space_onboarding`, только завершённые строки) — счётчики по уровню/цели/роли/сфере + покрытие; вторичный `legacy_quiz` по «Точке А». Без данных отдельных участников |
| `telemetry_summary(days=30)` | Агрегаты продуктовой телеметрии за окно `days` (1..30): счётчики событий по имени/маршруту/дню + число активных участников. Только агрегаты, никогда не сырые строки |
| `feedback_summary(days=90)` | Агрегаты оценок уроков + NPS за окно `days` (1..90): средний балл, число ответов и детракторов по материалу, блок платформенного NPS, последние комментарии (PII, только founder) |
| `get_member_card(tg_user_id?, member_id?)` | Полный контекст ОДНОГО участника: личность, подписка, история уровней, ответы онбординга, «Точка А», фидбэк, NPS-drip и 30-дневный агрегат телеметрии. Передайте РОВНО ОДИН из `tg_user_id` / `member_id`; ответ — `{"found": true, …}` или `{"found": false}` |

Каждый вызов founder-инструмента дополнительно ограничен по частоте на участника
(sliding window); превышение возвращает `rate_limited`.

## Пример конфигурации клиента

### Claude Code (`.mcp.json`)

```json
{
  "mcpServers": {
    "edgelab-space": {
      "type": "http",
      "url": "https://mcp.edgelab.space/mcp",
      "headers": {
        "Authorization": "Bearer edgelabspace_YOUR_KEY"
      }
    }
  }
}
```

### Универсальный MCP-клиент

Любой клиент, поддерживающий remote MCP по Streamable HTTP, подключается тем же
URL и заголовком `Authorization`. Сервер работает stateless: каждый вызов —
самостоятельный HTTP-запрос, без отдельного хендшейка сессии.

## Проверка

После подключения попросите агента вызвать `list_library(limit=5)` — непустой
список материалов подтвердит, что ключ принят и подписка активна. Если сервер
отвечает `Unauthorized`, проверьте ключ и статус подписки.

> Нет MCP-клиента? Тот же сервер вызывается обычным `curl` по JSON-RPC —
> см. [CLI.md](CLI.md).
